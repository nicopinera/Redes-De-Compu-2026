Prácticamente todas las redes basan sus decisiones de encaminamiento en algún criterio de mínimo coste. El coste de las líneas se emplea como entrada principal para establecer las rutas óptimas.

Dada una red de nodos conectados por enlaces bidireccionales, donde cada enlace tiene un coste asociado (que puede variar según el sentido), se define el coste de una ruta como la suma de los costes de los enlaces atravesados. La mayoría de los algoritmos modernos son variantes de dos fundamentales: Dijkstra y Bellman-Ford.

\subsection{Algoritmo de Dijkstra}

El algoritmo de Dijkstra, también conocido como SPF (Shortest Path First), busca encontrar las rutas de mínimo coste desde un nodo origen específico hacia todos los demás nodos de la red. Este procedimiento esta dividido en etapas, construyendo los caminos en orden creciente de longitud.

Conceptos clave:
\begin{itemize}
	\item $N$: Conjunto de nodos de la red.
	\item $s$: Nodo origen único.
	\item $T$: Conjunto de nodos cuya ruta mínima ya es conocida (etiquetados permanentemente).
	\item $w(i,j)$: Coste directo del enlace de $i$ a $j$ ($ \infty $ si no están conectados).
	\item $L(n)$: Coste actual del camino más corto conocido desde $s$ hasta $n$.
\end{itemize}

El algoritmo itera siguiendo tres pasos principales hasta que todos los nodos se incorporan a $T$:

\subsubsection{Inicialización}
Se comienza con $T = \{s\}$. Para cada nodo $n \neq s$, el coste inicial $L(n)$ se establece como el peso del enlace directo desde el origen: $L(n) = w(s,n)$.

\subsubsection{Obtención del siguiente nodo}
Se selecciona un nodo $x$ que no pertenezca a $T$ y que posea el valor de $L(x)$ más pequeño. Este nodo $x$ se añade permanentemente al conjunto $T$.

\subsubsection{Actualización de los caminos}
Para todos los nodos $n \notin T$, se actualiza su coste comparando el valor actual $L(n)$ con la nueva ruta potencial a través de $x$:
$$ L(n) = \min[L(n), L(x) + w(x,n)] $$

\subsection{Algoritmo de Bellman-Ford}

El algoritmo de Bellman-Ford, base de los protocolos de vector-distancia, es un procedimiento descentralizado. A diferencia de Dijkstra, cada nodo solo requiere información de sus vecinos directos para converger a la solución óptima.

**Conceptos clave:**
\begin{itemize}
	\item $s$: Nodo origen.
	\item $h$: Número máximo de enlaces (saltos) permitidos en la iteración actual.
	\item $L_h(n)$: Coste del camino mínimo desde $s$ hasta $n$ utilizando como máximo $h$ enlaces.
\end{itemize}

\subsubsection{Inicialización}
Para $h=0$, el coste al origen es 0 ($L_0(s)=0$) y para cualquier otro nodo $n$, el coste es infinito ($L_0(n)=\infty$).

\subsubsection{Actualización Iterativa}
En cada etapa $h+1$, el algoritmo calcula un nuevo coste comparando todas las rutas posibles a través de los vecinos $j$:
$$ L_{h+1}(n) = \min_j [L_h(j) + w(j,n)] $$

Este proceso se repite hasta que no se encuentran mejoras en los costes, garantizando que se ha hallado el camino óptimo (siempre que no existan ciclos de coste negativo).
